LDAP Management module for Hubot

Created a management module for LDAP, using Hubot robotics for the Slack messaging service. Code snippet can be found here, although requires the full repository to work (intellectual property not owned by me).

The code uses the Python LDAP module and GPG credentials for basic user management, e.g. creating and modifying users and groups.

Whilst many LDAP tools already exist, this solution suited us for a number of reasons:  

  • Slack provides a message history, allowing us to check tasks that had been done;
  • Having this functionality in our messaging service made staff onboarding easier; and
  • Using encrypted credentials allowed for a more secure environment, whilst allowing more staff members to perform basic tasks.

This project was very engaging, as I’m a beginner at Python programming, and had a tangible and satisfying end result. I enjoyed this project so much, that I chose it for the topic of presentation for the 2016 eResearch Australasia conference. Poster design was done by our Marketing Team, and can be viewed here.

Puppet overhaul for HPC cluster management

One of my challenges at my past employment was to assess and redesign the configuration management for the HPC cluster. The system was using Puppet and Kickstart to manage system files and reinstallation, but with multiple collaborators and no version control had become unwieldly.

I was able to create a new Puppet repository, using GitHub for version control, and utilised Puppet’s recommended roles and profiles as well as Hiera data to allow for dynamic values such as IP addresses and zoning configuration. The new solution kept Kickstart as the method of rebuilding nodes as required, and through a Puppet autosign configuration rebuilds were achieved from a single command prompt.

This project, although large, was very rewarding as I have a natural penchant for organisation and lists, which is well suited to configuration management.
